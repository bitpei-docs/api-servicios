Inicio Actualizado: 2019-06-01

API diseñada para la venta de servicios y servicios financieros proporciona una plataforma robusta y segura para que los usuarios puedan explorar, adquirir y gestionar una variedad de servicios, incluyendo servicios financieros.

Login()

Este endpoint permite autenticar a un usuario mediante el envío de credenciales a través de una solicitud POST a la ruta proporcionada.

POST /servicios-dev.bitpei.mx/login

Parámetros: {Datos}: Serie de datos requeridos para obtener acceso en el servicio de la API. Este parámetro es dinámico y debe ser proporcionado en el cuerpo de la solicitud.

Respuesta Exitosa: Código de estado HTTP 200 OK. Cuerpo de la respuesta en formato JSON, que contiene información detallada sobre el acceso a la API y el token requerido.

Informacion requerida para el obtener el token de acceso a la API.

200

La transaccion fue aplicada correctamente, se devuelve la respuesta

400

Ocurrio algun error al procesar la solicitud, debera intentar nuevamente.

Catalogo

Seccion con los metodos para consultar la informacion del catalogo de productos, subcatalogosy parametros de ubicacion para la aplicacion de ventas y transacciones.

Productos(string)

Este endpoint permite obtener información sobre productos asociados a una terminal específica mediante una solicitud GET. La ruta incluye un parámetro dinámico, {Terminal_Id}, que deberá ser reemplazado por el identificador único de la terminal para la cual se desea recuperar la información de productos.

GET /api/catalogos/productos/{Terminal_Id}

Parámetros: {Terminal_Id}: Identificador único de la terminal para la cual se desea obtener información sobre productos. Este parámetro es dinámico y debe ser proporcionado en la URL.

Respuesta Exitosa: Código de estado HTTP 200 OK. Cuerpo de la respuesta en formato JSON, que contiene información detallada sobre los productos asociados a la terminal especificada.

cURL para consultar los productos asignados al numero de terminal proporcionado.

Parametros()

Endpoint que proporciona acceso a un catálogo de parámetros a través de una solicitud GET. Este catálogo de parámetros contiene información clave que se utiliza en la configuración y funcionamiento de las transacciones. La respuesta de esta solicitud proporcionará una lista de parámetros junto con sus valores correspondientes.

GET /api/catalogos/parametros

Parámetros: No se especifica ningún parámetro en la URL, ya que se trata de una solicitud GET.

Respuesta Exitosa: Código de estado HTTP 200 OK. Cuerpo de la respuesta en formato JSON, que incluye la lista de parámetros y sus valores asociados.

cURL para consultar el catalogo de parametros, el resultado esta relacionado con el producto y subproducto donde debera ser visualizado esta informacion.

Ubicacion()

Seccion que contiene los metodos para la consulta de datos para la direccion del comisionista como entidad federativa, municipio y colonia.

GET /api/catalogo/entidad-federativa

cURL para la consulta del catalogo de entidades federativas, el id_entidad_federativa debera ser proporcionado en el registro del domicilio del cajero.

GET /api/catalogo/municipio?idEntidadFederativa={ID_ENTIDAD_FEDERATIVA}

cURL para realizar la consulta de los municipios por entidad federativa, el id_municipio debera ser proporcionado en el registro del domicilio del cajero.

GET /api/catalogo/colonia?IdMunicipio={ID_MUNICIPIO}

cURL para realizar la consulta de colonias por id_municipio, el id_colonia debera ser proporcionado para el registro de domicilio de los cajeros

Cajero

Metodos para el registro/bajas de cajeros.

Registrar(DatosPersonales)

Endpoint para el registro de un nuevo cajero, para el cual solicita datos personales del comisionista, del negocio y ubicacion, al completar el registro se generara el terminal_id que es utilizado en la ruta /api/transaccion/aplicar. El numero de terminal_id sera retornado en la propiedad mensaje.

POST /api/cajero/registrar

Parámetros: {DatosPersonales}: Serie de datos para la cual se desea registrar en el servicio de la API. Este parámetro es dinámico y debe ser proporcionado en el cuerpo de la solicitud.

Respuesta Exitosa: Código de estado HTTP 200 OK. Cuerpo de la respuesta en formato JSON, que contiene información detallada sobre el estatus asociados a la transaccion especificada.

cURL para el registro de nuevos cajeros.

Obtener()

Endpoint que al consultar mediante una solicitud GET proporciona una lista detallada de cajeros activos conectados a la Api, su información, ingresos y egresos.

GET /api/cajero/obtener

Parámetros: No se especifica ningún parámetro en la URL, ya que se trata de una solicitud GET.

Respuesta Exitosa: Código de estado HTTP 200 OK. Cuerpo de la respuesta en formato JSON, que incluye la lista de cajeros activos y sus valores asociados.

cURL para consultar la lista de cajeros registrados por el comisionista.

Baja(int)

Endpoint que se utiliza para realizar la baja o eliminación de un cajero específico identificado por su ID_CAJERO en el sistema. Esta operación implica retirar el cajero de la base de datos o desactivarlo, lo que puede ser necesario en situaciones como la retirada física del cajero, cambios en la configuración del sistema, o por cualquier otra razón que requiera la eliminación del registro del cajero. Para realizar esta acción, se espera que el cliente proporcione el ID_CAJERO como parámetro en la URL.

DELETE /api/cajero/baja/{ID_CAJERO}

Parámetros: {ID_CAJERO} Identificador unico del cajero el cual se requiere dar de baja. Este parámetro es dinámico y debe ser proporcionado en la URL.

Respuesta Exitosa: Código de estado HTTP 200 OK. Cuerpo de la respuesta en formato JSON, que incluye datos del personales del cajero.

cURL para dar de baja al cajero registrado por el comisionista.

Servicio

Seccion que contiene los metodos necesarios que usa el comisionista pueda consultar informacion sobre el estado del servicio y el saldo disponible.

Saldo()

Este método proporciona información sobre el saldo disponible del comisionista, así como el monto total de ingresos y egresos transaccionados durante el día de operación. La línea de autorización disponible para el comisionista dependerá de la configuración asignada..

GET /api/servicio/saldo

Parámetros: No se especifica ningún parámetro en la URL, ya que se trata de una solicitud GET.

Respuesta Exitosa: Código de estado HTTP 200 OK. Cuerpo de la respuesta en formato JSON, que incluye el estatus del servicio e informacion del saldo del comisionista.

cURL para la consulta de saldo del comisionista.

Reporte

Seccion que contiene metodos para la generacion de reportes por fecha indicada.

Reporte(string)

Metodo para la generacion del reporte por fecha, el cual contiene la informacion de cada transaccion realizada en la fecha indicada y por cada comisionista.

GET /api/reporte/reporte/{fecha}

Parámetros: {fecha}: Fecha para la cual se desea obtener información sobre el informe de transacciones. Este parámetro es dinámico y debe ser proporcionado en la URL.

Respuesta Exitosa: Código de estado HTTP 200 OK. Cuerpo de la respuesta en formato JSON, que contiene información detallada sobre las transacciones realizadas y el detalle de cada una.

cURL para la consulta de transacciones aplicadas por fecha.
