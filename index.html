Inicio Actualizado: 2019-06-01

API diseñada para la venta de servicios y servicios financieros proporciona una plataforma robusta y segura para que los usuarios puedan explorar, adquirir y gestionar una variedad de servicios, incluyendo servicios financieros.

Login()

Este endpoint permite autenticar a un usuario mediante el envío de credenciales a través de una solicitud POST a la ruta proporcionada.

POST /login

Informacion requerida para el obtener el token de acceso a la API.

200

La transaccion fue aplicada correctamente, se devuelve la respuesta

400

Ocurrio algun error al procesar la solicitud, debera intentar nuevamente.

404

La conexion no fue aplicada.

Catalogo

Seccion con los metodos para consultar la informacion del catalogo de productos y subcatalogos para la aplicacion de ventas y transacciones.

Productos(string)

Este endpoint permite obtener información sobre productos asociados a una terminal específica mediante una solicitud GET. La ruta incluye un parámetro dinámico, {Terminal_Id}, que deberá ser reemplazado por el identificador único de la terminal para la cual se desea recuperar la información de productos.

Parámetros: {Terminal_Id}: Identificador único de la terminal para la cual se desea obtener información sobre productos. Este parámetro es dinámico y debe ser proporcionado en la URL.
Respuesta Exitosa: Código de estado HTTP 200 OK. Cuerpo de la respuesta en formato JSON, que contiene información detallada sobre los productos asociados a la terminal especificada.

GET /api/catalogos/productos/{Terminal_Id}

Parametros()

Endpoint que proporciona acceso a un catálogo de parámetros a través de una solicitud GET. Este catálogo de parámetros contiene información clave que se utiliza en la configuración y funcionamiento de las transacciones. La respuesta de esta solicitud proporcionará una lista de parámetros junto con sus valores correspondientes.

Parámetros: No se especifica ningún parámetro en la URL, ya que se trata de una solicitud GET.

Respuesta Exitosa: Código de estado HTTP 200 OK. Cuerpo de la respuesta en formato JSON, que incluye la lista de parámetros y sus valores asociados.

GET /api/catalogos/parametros

Cajero

Metodos para el registro/bajas de cajeros.

Registrar(DatosPersonales)

Registra un nuevo cajero, al completar el registro se generara el terminal_id que es utilizado en la ruta /api/transaccion/aplicar. El numero de terminal_id sera retornado en la propiedad mensaje.

POST /api/cajero/registrar

cURL para el registro de nuevos cajeros.

Productos()

Este método proporciona la lista detallada de cajeros activos conectados a la Api, su información, ingresos y egresos.

GET /api/cajero/obtener

cURL para consultar la lista de cajeros registrados por el comisionista.

Baja(int)

/api/cajero/baja/{idCajero}

Servicio

Seccion que contiene los metodos necesarios para el comisionista consulte la informacion sobre el estado del servicio y el saldo disponible.

Saldo()

GET /api/servicio/saldo

Este método proporciona información sobre el saldo disponible del comisionista, así como el monto total de ingresos y egresos transaccionados durante el día de operación. La línea de autorización disponible para el comisionista dependerá de la configuración asignada..

Transaccion

Seccion que contiene los metodos para la aplicacion de transacciones y consulta de estatus de la operacion.

ConfirmarTransaccion(string)

Este endpoint permite confirmar y puede ser consumido para validar el estatus de una transacción mediante una solicitud GET. El parámetro dinámico {idTransaccion} debe ser reemplazado por el identificador único de la transacción que se desea confirmar.

GET /api/transaccion/confirmar/{idTransaccion}

AplicaTransaccion(SolicitudTransaccion)

Endpoint para aplicar los servicios que son devueltos en el catalogo de Productos, para cada solicitud debera enviar un encabezado Signature que contendra la firma del cuerpo del mensaje.

POST /api/transaccion/aplicar

200

La transaccion fue aplicada correctamente, se devuelve la respuesta original de la operacion

400

Ocurrio algun error al procesar la solicitud, debera intentar nuevamente.

404

La transaccion no fue aplicada, se devuelve la respuesta original de la operacion.

Reporte

Seccion que contiene metodos para la generacion de reportes por fecha indicada.

Reporte(string)

Metodo para la generacion del reporte por fecha, el cual contiene la informacion de cada transaccion realizada en la fecha indicada y por cada comisionista.

GET /api/reporte/reporte/{fecha}

Consultas

Seccion que contiene los metodos para la consulta de datos para la direccion del comisionista como entidad federativa, municipio y colonia.

Consulta

GET /api/catalogos/entidad-federativa

cURL para la consulta del catalogo de entidades federativas, el id_entidad_federativa debera ser proporcionado en el registro del domicilio del cajero.

GET /api/catalogos/municipio

cURL para realizar la consulta de los municipios por entidad federativa, el id_municipio debera ser proporcionado en el registro del domicilio del cajero.

GET /api/catalogos/colonia

cURL para realizar la consulta de colonias por id_municipio, el id_colonia debera ser proporcionado para el registro de domicilio de los cajeros
